# 小说工作台使用规范与整体架构

## 1. 概览

这套小说工作台不是简单的写作编辑器，而是一套带有 AI 生成、版本管理、审校、市场分析、平台适配和连续性修复能力的小说生产系统。

它的核心原则是：

1. 先建基础
2. 再写正文
3. 最后做修复和商业判断
4. 所有生成结果都要可落库、可回读、可回滚

---

## 2. 核心工作流

### 2.1 进入项目

进入项目时只做一件事：

- 读取本地数据库中已经保存的数据

不会做这些事：

- 不自动调用大模型
- 不自动重新规划
- 不自动重新生成正文
- 不自动重新跑市场分析或平台适配

进入时会读取的内容包括：

- 项目基础信息
- 世界观
- 角色
- 大纲
- 章节
- 章节版本
- 审校记录
- 运行记录
- 市场审计结果
- 平台适配结果

### 2.2 先做基础层

基础层是整本小说的地基，包含：

- 世界观
- 角色
- 大纲

推荐顺序：

1. 生成世界观
2. 生成角色
3. 生成大纲

这三项决定：

- 故事发生在哪个世界
- 谁在推动故事
- 故事怎么展开

### 2.3 再做写作层

写作层是正文生产阶段，包含：

- 当前章节编辑
- 当前章节正文生成
- Agent 链执行

正文生成不是孤立发生的，它会依赖：

- 世界观
- 角色
- 大纲
- 上一章正文
- 当前章目标
- 当前章冲突
- 当前章钩子

### 2.4 最后做后处理层

正文之后再做：

- 连续性修复
- 市场审计
- 平台适配

这三项是创作后的优化和判断，不是基础创作本身。

---

## 3. 模块职责

### 3.1 项目管理模块

功能：

- 创建项目
- 查看项目
- 删除项目
- 切换项目
- 选择写作模型

项目保存的信息：

- 标题
- 题材
- 目标读者
- 风格标签
- 商业标签
- 状态

---

### 3.2 基础创作模块

#### 世界观

保存：

- 世界摘要
- 规则
- 时间锚点
- 未知项
- 世界结构信息

作用：定义故事世界的基本规则。

#### 角色

保存：

- 角色名
- 角色定位
- 原型
- 动机
- 目标
- 冲突
- 关系信息
- 成长线

作用：定义故事由谁推动。

#### 大纲

保存：

- 总纲
- 卷纲
- 章纲
- 转折点
- 冲突点
- 钩子
- 父子结构

作用：定义故事怎么展开。

---

### 3.3 写作模块

#### 章节管理

每章保存：

- 章节序号
- 标题
- 章节目标
- 章节摘要
- 冲突
- 结尾钩子
- 正文
- 所属大纲

#### 正文生成

当前按钮语义已经统一为：

- `生成正文`

正文生成时会带上：

- 世界观
- 角色
- 大纲
- 上一章正文

结果会：

- 写回章节正文
- 写入版本历史
- 写运行记录

#### 手动编辑正文

可以直接编辑正文。系统会自动：

- 保存章节
- 留版本
- 支持回滚

#### 版本历史

可查看：

- 当前版本
- 历史版本
- 来源
- 版本差异
- 回滚

版本来源包括：

- 手动编辑
- Agent 写回
- 修复生成
- 回滚产生

---

### 3.4 Agent 模块

#### 基础规划 Agent

按钮语义：

- `生成基础规划`

作用：只规划三件事：

- 世界观
- 角色
- 大纲

不做什么：

- 不直接写正文
- 不默认进入后处理

#### 正文生成 Agent

作用：根据基础层 + 上一章正文生成当前章正文。

#### 完整 Agent 链

作用：执行一条综合生成链路，可能包括：

- 规划
- 正文
- 审校
- 修复

#### 连续性修复 Agent

按钮：

- `连续性修复`

作用：修复：

- 世界观冲突
- 角色前后不一致
- 章节逻辑断裂
- 节奏问题

#### 市场审计 Agent

作用：判断作品的商业潜力：

- 是否适合上架
- 风险在哪里
- 建议怎么改

保存位置：

- `reviews`
- `review_type = market_review`

#### 平台适配 Agent

作用：判断作品是否适合特定平台：

- 节奏适配
- 钩子适配
- 受众适配
- 结构适配

保存位置：

- `reviews`
- `review_type = platform_fit`

## 4. 数据流

### 4.1 启动项目

前端请求后端读取本地数据。

### 4.2 建基础

先生成：

- 世界观
- 角色
- 大纲

### 4.3 写正文

正文生成时会带上：

- 世界观
- 角色
- 大纲
- 上一章正文

### 4.4 版本保存

每次修改正文会自动保存版本。

### 4.5 后处理

可单独做：

- 连续性修复
- 市场审计
- 平台适配

### 4.6 再次进入

直接读取本地保存结果，不重新跑模型。

---

## 5. 整体架构

### 5.1 前端 UI 层

主要文件：

- `ui/web/src/pages/NovelProjectWorkspace.tsx`

负责：

- 展示工作台
- 展示基础层
- 展示正文区
- 展示审校 / 版本 / 市场 / 平台
- 发起生成请求

### 5.2 API 路由层

主要文件：

- `ui/server/src/routes/novel.ts`

负责：

- 接收前端请求
- 调用生成逻辑
- 写回存储
- 返回结果

### 5.3 存储层

主要文件：

- `ui/server/src/novel.ts`

负责：

- 项目、世界观、角色、大纲、章节、版本、审校、运行记录的读写
- SQLite 优先
- 没有数据库时回退 JSON

### 5.4 AI / LLM 层

主要目录：

- `ui/server/src/llm/*`

负责：

- 规划
- 正文生成
- Agent 执行
- 修复
- 平台适配

## 6. 使用建议

### 标准流程

1. 创建项目
2. 生成基础规划
   - 世界观
   - 角色
   - 大纲
3. 检查并手动调整基础层
4. 选择章节并生成正文
5. 逐章编辑、回滚、修复
6. 做市场审计和平台适配

### 推荐原则

- 基础层先行
- 正文依赖上下文
- 后处理放在正文之后
- 所有生成结果必须落库
- 再次进入时必须优先读取已保存数据

---

## 7. 设计思想总结

这套工作台的设计思想可以概括为：

> 本地持久化优先、基础层先行、正文依赖上下文、后处理可审计。

它不是一个单纯的聊天式写作工具，而是一套面向小说生产的 AI 工作流系统。

---

## 8. 一句话总结

> 这套小说工作台是一个“先建基础、再写正文、最后做修复与商业判断”的 AI 小说创作系统，所有结果都可持久化、可回读、可回滚。
