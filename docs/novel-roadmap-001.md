# 小说引擎规划 v0.1

## 目标

将“自动写小说”能力纳入 MangaForge Studio 的内容生产链路，形成：

- 小说生成
- 分镜生成
- 无限画布编排
- 漫剧生产

的完整上游引擎。

---

## 当前认知

### 已有基础
- 分镜基础已存在
- 无限画布基础已存在
- 前端资产 / 画布 / Pipeline / StudioHome 已在整合
- `restored-src` 中已有成熟的多 agent / task / coordinator 设计可参考

### 需要新增
- 长篇小说规划
- 多 agent 小说协作
- 章节级连续性管理
- 商业热门题材吸收

## 推荐的实现分层

### A. 作品层
一本小说一个独立作品对象，负责：
- 标题
- 类型
- 字数目标
- 受众
- 风格
- 主题
- 商业标签

### B. 世界观层
负责：
- 规则
- 势力
- 地点
- 技术体系
- 能力体系
- 时间线

### C. 角色层
负责：
- 人设
- 动机
- 关系图
- 成长线
- 已知信息
- 行为约束

### D. 结构层
负责：
- 总纲
- 卷纲
- 幕纲
- 章纲
- 场景纲

### E. 生成层
负责：
- 正文生成
- 章节续写
- 章节修订
- 风格统一

### F. 校验层
负责：
- 连续性检查
- 设定冲突检查
- 节奏检查
- 伏笔回收检查

## 多 Agent 角色建议

### 1. 主编 Agent
职责：
- 定位作品类型
- 把控商业方向
- 监督整体质量

### 2. 世界观 Agent
职责：
- 构建世界规则
- 维护设定文档
- 提供扩展空间

### 3. 人物 Agent
职责：
- 创建角色卡
- 维护关系网
- 追踪角色成长

### 4. 大纲 Agent
职责：
- 生成总纲
- 生成卷纲
- 生成章纲

### 5. 章节 Agent
职责：
- 细化每章冲突
- 组织章节事件链
- 生成可执行章节内容

### 6. 连续性审校 Agent
职责：
- 发现前后矛盾
- 检查伏笔回收
- 检查人物行为一致性

### 7. 市场分析 Agent
职责：
- 总结热门类型特征
- 提炼受欢迎结构
- 为新作品提供策略输入

### 8. 风格统一 Agent
职责：
- 统一文风
- 统一节奏
- 统一叙事视角

## 长篇实现关键

### 1. 分层记忆
长篇不能依赖单次上下文，必须有：
- 世界观摘要
- 角色摘要
- 事件摘要
- 伏笔摘要
- 时间线摘要

### 2. 章节索引
每章都应有：
- 目标
- 冲突
- 结果
- 钩子
- 分镜输入

### 3. 状态持久化
每次生成后要写入：
- 作品状态
- 角色状态
- 时间线状态
- 伏笔状态

### 4. 版本化
支持：
- 大纲版本
- 章节版本
- 修订版本
- 不同风格分支

## 第一阶段建议

不要直接追求“百万字全文一口气生成”。

先做：
1. 作品规划
2. 世界观与角色卡
3. 总纲 / 卷纲 / 章纲
4. 章节生成与校验
5. 输出到分镜结构

---

## 第二阶段建议

加入：
- 热门题材吸收
- 爆款结构总结
- 章节节奏优化
- 多作品知识复用

---

## 第三阶段建议

与漫剧流程打通：

- 小说章节自动切场景
- 场景自动转分镜
- 分镜自动进无限画布
- 无限画布自动驱动漫剧生成
