# 小说工作台前端规范 v0.1

## 目标

为小说引擎提供一套清晰、可扩展、适合长篇创作的前端工作台界面，支撑：

- 小说项目管理
- 题材与作品规划
- 世界观编辑
- 角色关系维护
- 大纲 / 章节生成与修订
- 连续性审校
- 小说到分镜的结构化输出
- 运行记录与状态回看

---

## 设计原则

1. **分层呈现**
   - 让用户先看作品，再看世界观，再看角色，再到大纲与章节
   - 不把所有内容堆在一个超长表单里

2. **长篇友好**
   - 支持卷 / 章 / 场景级浏览
   - 支持状态压缩与摘要展示
   - 支持快速定位和跳转

3. **面向协作**
   - 页面要能容纳多 agent 协作结果
   - 显示“谁生成了什么”
   - 显示“当前状态 / 建议 / 修订记录”

4. **面向下游**
   - 前端不仅是写小说，还要输出给分镜和无限画布
   - 页面必须支持结构化导出

5. **可追溯**
   - 每次生成和修订都要留痕
   - 允许回滚与对比

## 建议页面结构

### 1. 小说项目大厅

用途：
- 查看所有小说项目
- 创建、删除、进入项目
- 展示项目状态、字数目标、当前进度

展示信息：
- 标题
- 题材
- 目标字数
- 当前字数
- 状态
- 最近运行记录

---

### 2. 作品规划页

用途：
- 设置作品定位
- 选择题材
- 选择长度目标
- 定义风格标签
- 接收市场分析结果

展示信息：
- 题材建议
- 商业标签
- 风格标签
- 受众画像
- 创作策略

---

### 3. 世界观页

用途：
- 编辑世界规则
- 维护势力、地点、系统
- 查看世界观版本历史

展示信息：
- 世界摘要
- 规则列表
- 势力列表
- 地点列表
- 系统设定
- 时间锚点

---

### 4. 角色页

用途：
- 创建与维护角色卡
- 管理角色关系图
- 查看角色成长线
- 记录角色当前状态

展示信息：
- 角色名
- 类型
- 动机
- 目标
- 当前状态
- 隐秘信息
- 关系图

---

### 5. 大纲页

用途：
- 管理总纲 / 卷纲 / 章纲
- 支持树形结构浏览
- 支持单节点修订

展示信息：
- 结构树
- 节点摘要
- 冲突点
- 转折点
- 钩子
- 目标长度

---

### 6. 章节页

用途：
- 查看章节正文
- 查看章节结构化信息
- 一键生成 / 继续 / 修订
- 章节与分镜联动

展示信息：
- 章节标题
- 章节目标
- 章节摘要
- 场景列表
- 冲突点
- 结尾钩子
- 章节正文

---

### 7. 审校页

用途：
- 检查一致性
- 检查人物、时间线、伏笔
- 显示修订建议

展示信息：
- 设定冲突
- 时间线冲突
- 角色行为冲突
- 伏笔回收情况
- 审校结果

---

### 8. 分镜输出页

用途：
- 将章节转换成分镜输入
- 输出场景切分结果
- 对接无限画布

展示信息：
- 场景列表
- 视觉关键词
- 情绪曲线
- 镜头建议
- 画面提示词

---

### 9. 运行记录页

用途：
- 查看每次生成 / 修订 / 审校的执行历史
- 追踪耗时、输入、输出、错误

展示信息：
- 运行类型
- 步骤
- 状态
- 耗时
- 结果引用
- 错误信息

---

- 小说项目
- 作品规划
- 世界观
- 角色
- 大纲
- 章节
- 审校
- 分镜输出
- 运行记录

## UI 风格建议

### 视觉风格
- 继续保持 MangaForge Studio 现有的现代化风格
- 使用卡片、分栏、树形结构、时间线、标签、状态徽标
- 适配长篇内容浏览

### 交互建议
- 支持分栏编辑
- 支持折叠摘要层
- 支持快速跳转
- 支持版本对比
- 支持局部生成与局部重写

## 与后端 API 的映射

前端页面应尽量对应 API 分层：

- `Projects` ↔ 小说项目 CRUD
- `Planning` ↔ 市场分析 / 作品定位
- `Worldbuilding` ↔ 世界观 CRUD
- `Characters` ↔ 角色 CRUD
- `Outline` ↔ 大纲生成
- `Chapters` ↔ 章节生成 / 续写 / 修订
- `Review` ↔ 审校接口
- `Storyboard` ↔ 分镜输出接口
- `Runs` ↔ 运行记录接口
